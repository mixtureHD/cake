由于最近换了系统，博客环境没了重新搭建了一下环境导致`hexo d` 一直报错，花了几个小时的时间调试，结果发现只是网络问题。特此写博客记录！！



<!--more-->



知道真相的我简直有点受不了。



先介绍一下背景，**如果没兴趣看的，可直接跳到下面出错图片查看是否和我报错一样。**



**背景**

   	1. 重装了系统
     2. 路由搭了 `ss/ssr`服务
      3. 使用的 `移动宽带` 



重装了系统导致博客环境没了需要重新搭建。路由搭了`ss`所以进入`github.com`正常（后面把ss关了之后根本进不去）。使用的移动宽带！移动宽带！移动宽带！（强烈怀疑是移动宽带问题）。



## 错误

在执行了 `hexo clean` `hexo g` 之后准备使用`hexo d`推送到github的时候就发生了错误，一直告诉我ssh有问题。不断的添加删除`github`上的ssh公钥发现还是报这个错误。**且执行`ssh -T git@github.com`报错** 

![github提交失败](/img/blog/error.png)



这个时候建议**把所有fan墙软件关闭**之后，再试试能不能打开 [github](https://github.com/) 你就会发现很可能**进不去！！！** 然后你掏出你的手机输入这个网址(断wifi，用数据)，你会发现奇迹的进去了!!!



## 解决

**电脑拔掉网线，手机开启热点，电脑wifi链接，请再试一次 `hexo d`**



你会发现好了！好了！！好了！！！
